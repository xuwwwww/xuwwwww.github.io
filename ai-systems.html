What PerfRunbench Does PerfRunbench 能做什麼

Resource Guard

Wrap a training or benchmark command with memory budget, reserved CPU cores, CPU quota, cgroup sampling, and optional hard kill 包住原本的 training 或 benchmark command，加入 memory budget、reserved CPU cores、CPU quota、cgroup sampling 與 optional hard kill

CPU / Memory Tuning

Compare Linux, WSL, Windows, runtime, and memory profiles against a baseline with repeat-aware reports 把 Linux、WSL、Windows、runtime、memory profiles 跟 baseline 比較，並輸出 repeat-aware reports

GPU Tuning

Apply reversible NVIDIA runtime profiles such as performance, balanced, and guard, then compare GPU throughput and TFLOPS 套用可還原的 NVIDIA runtime profiles，例如 performance、balanced、guard，並比較 GPU throughput 與 TFLOPS

Reports

Generate HTML reports and recommendation JSON so runs are inspectable without digging through raw terminal logs 產生 HTML reports 與 recommendation JSON，讓結果不用翻 raw terminal logs 也能檢查

02

How It Works 如何做到

1

Wrap the existing command 包住原本的 command

PerfRunbench does not replace the training script, it launches the original command after --, so the workload can stay in its own Python, conda, venv, shell, or Docker environment PerfRunbench 不取代 training script，而是在 -- 後啟動原本的 command，因此 workload 可以保留自己的 Python、conda、venv、shell 或 Docker 環境 

autotuneai run --memory-budget-gb 22 -- python train.py
2

Select an executor 選擇 executor

The runner chooses local, systemd, or docker, on Linux/WSL, systemd can create a transient scope so CPU and memory limits are enforced through cgroups Runner 會選擇 local、systemd 或 docker，在 Linux/WSL 上，systemd 可以建立 transient scope，透過 cgroups 執行 CPU 與 memory limits

3

Apply resource guard 套用 resource guard

CPU quota, reserved cores, memory budget, affinity, and optional hard-kill rules are applied around the process, when cgroups are available, it samples memory.current, memory.peak, and cpu.stat 在 process 外層套用 CPU quota、reserved cores、memory budget、affinity 與 optional hard-kill，若可使用 cgroups，會取樣 memory.current、memory.peak 與 cpu.stat

4

Apply runtime tuning 套用 runtime tuning

System profiles adjust reversible OS/runtime knobs, runtime profiles set process-local environment variables, GPU profiles use nvidia-smi controls such as persistence mode, power limit, and supported application clocks when available System profiles 會調整可還原的 OS/runtime knobs；runtime profiles 會設定 process-local environment variables；GPU profiles 會在可用時透過 nvidia-smi 控制 persistence mode、power limit 與 supported application clocks

5

Compare against paired baselines 用 paired baselines 比較

Performance sweeps run baseline controls near candidate runs, then rank profiles by normalized throughput such as samples_per_second or gpu_tflops_estimate, this avoids judging candidates only by cold-start raw numbers Performance sweeps 會在 candidate 附近插入 baseline controls，再用 normalized throughput，例如 samples_per_second 或 gpu_tflops_estimate 排名，避免只看 cold-start raw numbers

6

Restore and write evidence 還原並輸出佐證

Runs write before/after/diff files such as system_tuning_before.json, gpu_tuning_diff.json, resource_summary.json, and HTML reports, interrupted runs can be recovered with autotuneai restore --active 每次 run 會輸出 before/after/diff 檔案，例如 system_tuning_before.json、gpu_tuning_diff.json、resource_summary.json 與 HTML reports，中斷後可用 autotuneai restore --active 復原

Core Commands 核心指令

Run with CPU and memory guard 使用 CPU 與 memory guard 執行 

autotuneai run --executor systemd --memory-budget-gb 22 --cpu-quota-percent 90 -- python train.py

Compare runtime tuning 比較 runtime tuning 

autotuneai compare-tuning --profile linux-throughput --runtime-profile runtime-cpu-performance --repeat 3 -- python benchmark.py

Run GPU performance sweep 執行 GPU performance sweep 

autotuneai optimize-performance --target gpu --gpu-profile nvidia-performance -- python examples/gpu_training_pressure.py

Restore runtime settings 還原 runtime settings 

autotuneai restore --active
